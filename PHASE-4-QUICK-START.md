## Deployment Options

### Option 1: Vercel (Recommended)
```bash
vercel --prod
# Follow prompts for domain configuration
```

**Pros**: Zero-config, automatic HTTPS, CDN, preview URLs  
**Cons**: Vendor lock-in

### Option 2: Docker
```bash
docker build -t ailydian-ultra-pro:gold .
docker run -d -p 3100:3100 --name ailydian ailydian-ultra-pro:gold
```

**Pros**: Portable, reproducible, scalable  
**Cons**: Requires infrastructure management

### Option 3: Manual VPS
```bash
# Extract to server
scp build/GOLD-DEPLOYMENT-*.tar.gz user@server:/var/www/
ssh user@server
cd /var/www && tar -xzf GOLD-DEPLOYMENT-*.tar.gz
pnpm i --prod
pm2 start ecosystem.config.js --env production
```

**Pros**: Full control, cost-effective  
**Cons**: Manual setup, maintenance overhead

## Troubleshooting

### Build Fails
```bash
# Clear cache and rebuild
rm -rf node_modules .pnpm-store
pnpm store prune
pnpm i
./gold-pipeline-web.sh
```

### Tests Fail
```bash
# Run in UI mode for debugging
pnpm test:ui

# Run specific test
pnpm test tests/smoke.spec.ts
```

### Deployment Issues
```bash
# Check Vercel status
vercel inspect ailydian-ultra-pro

# Re-deploy
vercel --force --prod
```

### Health Checks Fail
```bash
# Restart server
pm2 restart ailydian-ultra-pro

# Check logs
pm2 logs ailydian-ultra-pro --lines 100
```
